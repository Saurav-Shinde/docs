---
title: "Orders Management"
---

This module allows the admin to **track, process, and complete orders placed by brands (clients)**.

Whenever a client places an order, the system alerts the admin and provides step-by-step control until delivery is confirmed.

---

## New Order Notification

When a brand places a new order:

- A **red notification dot** appears on the Admin Dashboard
- It indicates which brand has placed a new order

To open orders:

1. Go to the **Admin Dashboard**
2. Find the brand in the onboarded brands list
3. Click **Manage** beside that brand
4. A **right-side drawer panel** opens
5. Navigate to the **Orders section**

Here the admin can see all orders placed by that brand.

---

## Viewing Order Details

Each order displays:

- Dish Name
- Quantity
- Order Amount
- Order Status

There is also a **View Recipe** button.

### View Recipe

When clicked, admin can see full breakdown of the dish:

- Ingredients used
- Quantity of each ingredient
- Cost per ingredient
- Total calculated cost

This helps admin verify production and costing accuracy before preparation.

---

## Processing an Order

Every order moves through the following stages.

---

### 1) Order Placed → Mark Preparing

After the kitchen/team starts preparing the order:

Click **Mark Preparing**

System will:

- Update order status to **Preparing**
- Show that work has started

The button will now change to:

**Mark Completed**

---

### 2) Preparing → Mark Completed

Once the order is ready and dispatched:

Click **Mark Completed**

System will:

- Mark order as completed
- Send notification to client that order is dispatched

---

### 3) Client Confirms Delivery

After receiving the order, the client clicks **Received** from their side.

Now admin will see beside the dish:

**Received**

This confirms delivery is successful.

---

## Deleting Completed Orders

After the order lifecycle is finished:

Admin can **Delete the order**

> Only completed orders should be deleted\
> This keeps dashboard clean and manageable

Deleting does NOT affect past costing records already used in reports.
